Day 1:
ho preparato i primi 3 capitoli di effective java cercando di applicare le nozioni apprese a java 17.
-static factory method
    creare un factory method statico per non usare costruttori permette di dare un nome alla creazione dell'oggetto, permette di gestire la creazione in modo piu libero, con la possibilitá di creare oggetti differenti in base ai parametri in entrata.

-Builder
    semplifica notevolmente la creazione di un oggetto, si utilizza in caso di oggetti complessi con numerosi parametri, riduce gli errori e aumenta la leggibilitá del codice

-singletone
    non l'ho utilizzato ma permette un istanza sola, va usato solo e solamente se si é certi della necessitá, un caso utile potrebbe essere una classe util, o una classe che valida varii tipi di dati o un logger.
